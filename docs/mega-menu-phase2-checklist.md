# Phase-2 Mega Menu — QA Checklist

목적
- Phase-2에서 도입된 메가메뉴(특히 `연구소 소개` 예외 처리)의 회귀 방지 및 운영 안정성 확보를 위한 검증 체크리스트이다.

검증 환경
- 로컬 개발 서버 또는 스테이징 환경에서 수행한다. 기본 호스트 예: `http://localhost:5173`.

핵심 확인 항목

1) `연구소 소개` 메뉴 품질 검사
- Hover 동작: 상단 네비게이션에서 `연구소 소개`에 마우스 오버한다.
  - 기대 결과: 하나의 메가메뉴 영역에 병합된 5개의 하위 항목이 동시에 한 화면에 모두 노출된다.

- `전체보기` 동작: 메가메뉴 내 `전체보기`(CTA)에서 클릭 또는 Enter 키 입력을 수행한다.
  - 기대 결과: 브라우저가 `/about/greeting`으로 이동하고, 메가메뉴가 정상적으로 닫힌다.

2) 하위 항목 이동 안정성
- 메가메뉴의 각 하위 항목을 클릭하거나 Enter 키로 선택한다.
  - 기대 결과: 해당 항목의 대상 경로로 정상 네비게이션되며, 메뉴가 닫힌다.

3) 비영향성(레거시 메뉴 보존)
- 다른 상위 메뉴들(메뉴 그룹)에 대해 hover/클릭을 수행한다.
  - 기대 결과: 기존 동작(레이아웃/지연/포털 동작 포함)에 회귀가 없어야 한다.

4) 빠른 상호작용 스트레스 테스트
- 빠른 hover 전환(마우스로 상위 메뉴를 연속으로 빠르게 이동) 및 연속 클릭(메뉴 항목을 빠르게 연타) 실행.
  - 기대 결과: 클릭 미이동, 이중 닫힘 등의 오동작이 없어야 한다.

검증 방법(수동)
- 위 항목들을 각 브라우저(Chrome, Edge, Firefox 권장)에서 수행하고, 콘솔에 에러가 없는지 확인한다.
- SPA 직접 접근(`/about/greeting`)도 테스트하여 라우팅이 정상 동작하는지 확인한다.

검증 기준
- 모든 항목 통과시 Phase-2 적용을 성공으로 본다.
- 실패 항목이 있으면 관련 리포트(재현 단계, 스크린샷, 콘솔 로그 포함)를 남긴다.

비고
- 이 체크리스트는 Phase-2 완료 기준 문서로 사용된다. 향후 코드 변경은 본 체크리스트를 반드시 통과해야 한다.
